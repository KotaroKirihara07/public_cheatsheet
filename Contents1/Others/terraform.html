Terraform

2025/04/26

----

概要

Docker、AWS、GCP、Azureの環境の構築

用語

HCL (HashiCorp Configuration Language) : Terraformでの設定ファイルを記述するための言語。

terraform.tfstate : Terraformが構築・管理している対象のインフラの状態を保存するためのファイルであり、ステートファイルと呼ばれる。このファイルを手動で削除したり、編集したりしてはいけない。

セットアップ

  1. 公式サイト からバイナリをダウンロードする。or パッケージマネージャーを利用してインストールする。
  2. terraform -vでバージョンを確認する。
  3. terraform -install-autocompleteでタブ補完機能を有効化する。

リソースの作成

手順

  1. プロジェクトディレクトリを作成して移動する
  2. 必要なファイルを作成し、編集する
  3. terraform initで初期化を行う
  4. terraform planで構築プランの確認を行う
  5. terraform applyで環境の構築を行う
  6. terraform destroyで環境の削除を行う

ファイル構成

ファイル構成はTerraform公式Style Guideに従う。

PROJECT_NAME
    ├ backend.tf
    ├ main.tf
    ├ outputs.tf
    ├ providers.tf
    ├ terraform.tf
    ├ terraform.tfvars
    ├ variables.tf
    ├ locals.tf
    ├ override.tf

providers.tf

providerブロックはproviderの設定を記述するためのブロック。

AWS用テンプレート

# Configure the AWS Provider
provider "aws" {
    region = "ap-northeast-1"
}

terraform.tf

terraformブロックはTerraform自体の設定や依存関係の設定を記述するブロック。

AWS用テンプレート

terraform {
    required_providers {
        aws = {
            source = "hashicorp/aws"
            version = "~> 5.0"
        }
    }
}

terraform.tfvars

変数に具体的な値を設定するためのファイル。

VARIABLE_NAME_1 = "hello"
VARIABLE_NAME_2 = 100
VARIABLE_NAME_3 = true

variables.tf

変数の定義を行うためのファイル。

variable "VARIABLE_NAME_1" {
    #変数の型
    type = string|number|bool|list(<TYPE>)|set(<TYPE>)|map(<TYPE>)|tuple([<TYPE>, ...])|object({<ATTR NAME> = <TYPE>, ... })
    #変数の説明
    description = "The description of the variable."
    #デフォルト値
    default =
    #値のバリデーション
    validation =
    #センシティブな値かどうかのフラグ
    sensitive = true|false
    #nullを許容するかどうかのフラグ
    nullable = true|false
}

outputs.tf

説明--------------------

output "VALUE_NAME_1" {
    description = "The description of the value"
    value =
}

main.tf

説明-----------------------------

#resourceブロック : 個々のインフラオブジェクトの設定を記述するためのブロック
resource "RESOURCE_TYPE_NAME" "LOCAL_RESOURCE_NAME" {
    KEY1 = VALUE1
    KEY2 = VALUE2
    KEY3 = VALUE3
}
