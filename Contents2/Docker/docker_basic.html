Docker

基本

2024/02/17

このページではDockerの基本についての説明を行う。

＊docker composeからdockerfileを使用してコンテナを作成することもできる

構造

LinuxOS上でDocker Engineを起動して、その上にコンテナを配置することができる。図で説明したほうがいい。

コード

用語

Docker Engine : コンテナを配置するためのソフトウェア。

コンテナ : LinuxOSっぽいものが入っており、そこでアプリケーションを立ち上げることができる。

Dockerイメージ : コンテナの設計書。コンテナからイメージを作成することもできる。

Docker Hub : 公式が運用しているDockerレジストリ(Dockerイメージが配布されている場所)である。

準備

Dockerで環境を構築する前のUbuntuにおける準備を以下のように行う。

  1. homeディレクトリ内に「workspace」というディレクトリを作成する
  2. 「workspace」ディレクトリ内に「docker_workspace」というディレクトリを作成する
  3. 「docker_workspace」ディレクトリ内にそれぞれのコンテナ用のディレクトリを作成する

dockerfileの使用方法

docker composeの使用方法 (バインドマウント方式)

  1. 「docker_workspace」ディレクトリ内にディレクトリ(dir_name)を作成する
  2. dir_name内に「volume」というディレクトリを作成する
  3. dir_name内に「docker-compose.yml」という名前のDocker Composeの定義ファイルを作成する
  4. docker compose -f home/workspace/docker_workspace/dir_name/docker-compose.yml up -d でコンテナを作成・起動する
  5. docker compose -f home/workspace/docker_workspace/dir_name/docker-compose.yml down でコンテナを停止・削除する
  6. でイメージを削除する
  7. でボリュームを削除する
