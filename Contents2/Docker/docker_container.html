このページではコンテナ管理を行うためのコマンドについての説明を行う。

docker、docker container、docker image、docker network、docker volume、docker commit、docker build、docker save、オプション

docker

単独コマンド。

docker コマンド

コマンド 説明
login Dockerレジストリにログインする
logout Dockerレジストリからログアウトする
search Dockerレジストリで検索する
version Docker Engineおよびコマンドのバージョンを表示する

docker container

コンテナ操作関連コマンド。

docker container 副コマンド オプション

副コマンド 説明
start コンテナを開始する
stop コンテナを停止する
create Dockerイメージからコンテナを作成する
run Dockerイメージをダウンロードし、コンテナを作成して起動する
rm 停止したコンテナを削除する
exec 実行中のコンテナ内でプログラムを実行する
ps コンテナ一覧を表示する
cp DockerコンテナとDockerホスト間でファイルをコピーする
commit Dockerコンテナをイメージに変換する

#コンテナを作成・起動

docker container [オプション] [イメージ名] [引数]

#コンテナを停止する

docker container stop [コンテナID/コンテナ名]

#すべてのコンテナを停止する

docker container stop $(docker container ps)

#コンテナを削除する

docker container rm [コンテナID/コンテナ名]

#起動しているコンテナ一覧を表示する

docker container ps

#存在するコンテナ一覧を表示する

docker container ps -a

#ポート指定してコンテナを作成する

docker container run -p 8080:80

#「net1」というネットワーク上にコンテナを作成する

docker container run -net=net1

#ホストからコンテナへファイルをコピーする

docker container cp [ホスト側パス] [コンテナ名:コンテナ側パス]

#コンテナからホストへファイルをコピーする

docker container cp [コンテナ名:コンテナ側パス] [ホスト側パス]

#コンテナ内を操作できるようにする

docker container run --name test1 -p 8080:80 httpd /bin/bash

#コンテナ内の操作を停止

exit

docker image

イメージ操作関連コマンド。

docker image 副コマンド オプション

副コマンド 説明
pull Docker Hubなどのリポジトリからイメージをダウンロードする
rm Dockerイメージを削除する
ps ダウンロードしたイメージ一覧を表示する
build Dockerイメージを作成する

#存在するイメージ一覧を表示する

docker image ps

#httpdイメージを削除する

docker image rm httpd

#すべてのイメージを削除する

docker image rm $(docker image ps)

#複数のイメージを削除する

docker image rm イメージ名1 イメージ名2 イメージ名3

docker network

ネットワーク操作関連コマンド。

docker network 副コマンド オプション

副コマンド 説明
connect コンテナをネットワークに接続する
disconnect コンテナをネットワークから切断する
create ネットワークをつくる
inspect ネットワークの詳細情報を表示する
ls ネットワークの一覧を表示する
prune 現在コンテナがつながっていないネットワークをすべて削除する
rm 指定したネットワークを削除する

#「net1」というネットワークを作成する

docker network create net1

docker volume

ボリュームマウントを行うためのコマンド

docker volume 副コマンド オプション

副コマンド 説明
create ボリュームをつくる
inspect ボリュームの詳細情報を表示する
ls ボリュームの一覧を表示する
prune 現在マウントされていないボリュームをすべて削除する
rm 指定したボリュームを削除する

ボリュームマウント

Docker Engineが管理している領域内にボリュームを作成して、ディスクとしてコンテナにマウントする。手軽に扱える半面、直接操作しづらい。

#手順

  1. docker volume create volume_name
  2. docker container run -v volume_name:コンテナの記憶領域パス

バインドマウント

Dcoker Engineの管理外であるOS上にボリュームを作成して、ディスクとしてコンテナにマウントする。直接操作しやすいという利点がある。

#手順

  1. OS上にマウント用のディレクトリを作成する
  2. docker container run -v OS上のディレクトリのパス:コンテナの記憶領域パス

docker commit

コンテナをイメージ化するためのコマンド。

docker commit コンテナ名 作成するイメージ名

docker build

Dockerfileからイメージを作成するためのコマンド。

docker build -t 作成するイメージ名 材料フォルダのパス

#手順

  1. 材料用ディレクトリを作成する
  2. 材料用ディレクトリに必要なものを格納する
  3. Dockerfileを作成する
  4. Dockerfileを材料用ディレクトリに格納する
  5. docker build -t 作成するイメージ名 材料フォルダのパス

docker save

作成したイメージをtarファイルにして持ち運び可能にするためのコマンド。

docker save -o ファイル名.tar 作成したイメージ名

オプション


オプション 説明
--name コンテナ名 コンテナ名を指定する
-p ホストのポート番号:コンテナのポート番号 ポート番号を指定する
-v ホストのディスク:コンテナのディレクトリ ボリュームをマウントする
--net=ネットワーク名 コンテナをネットワークに接続する
-e 環境変数=値 環境変数を指定する
-d バックグラウンドで実行する
-i コンテナに操作端末をつなぐ
-t 特殊キーを使用可能にする
-help 使い方を表示する
