DCL (Data Control Language)

2024/02/16

DCL (Data Control Language)はデータ制御言語と呼ばれ、データベースの変更を確定したり、取り消したりする。

COMMIT

#データベースに行った変更を確定する。

BEGIN TRANSACTION;
  DML statement1;
  DML statement2;
  DML statement3;
COMMIT;

#データベースの更新をまとめて確定する

BEGIN TRANSACTION;
  UPDATE table_name SET col_name1 = col_name1*1.08 WHERE col_name2 = 'note';
  UPDATE table_name SET col_name1 = col_name1 * 1.1 WHERE col_name2 = 'food';
COMMIT;

ROLLBACK

#データベースに行った変更を取り消す。

BEGIN TRANSACTION;
  DML statement1;
  DML statement2;
  DML statement3;
ROLLBACK;

#トランザクションをロールバックする

BEGIN TRANSACTION;
  UPDATE table_name SET col_name1 = col_name1*1.08 WHERE col_name2 = 'note';
  UPDATE table_name SET col_name1 = col_name1 * 1.1 WHERE col_name2 = 'food';
ROLLBACK;

GRANT

アクセス権限を定義するためのコマンド。データベースオブジェクトに対する権限の付与とロール内のメンバ資格の付与の二つにわけられる。

#すべてのユーザに特定のテーブルに対するSELECT権限を与える

GRANT SELECT ON Table_name TO PUBLIC;

#特定のユーザに特定のテーブルに対するすべての権限を与える

GRANT ALL PRIVILEGES ON Table_name TO user_name;

REVOKE

アクセス権限を剥奪するためのコマンド。

#すべてのユーザに特定のテーブルに対するSELECT権限を取り消す

REVOKE SELECT ON Table_name FROM PUBLIC;

#特定のユーザに特定のテーブルに対するすべての権限を取り消す

REVOKE ALL PRIVILEGES ON Table_name FROM user_name;
