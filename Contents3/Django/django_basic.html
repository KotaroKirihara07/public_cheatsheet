基本

2024/07/28

このページではDjangoの基本事項について説明する。

MVT

Model : データアクセス関係の処理を担当する

View : Modelとtemplateの仲介を行い、コントロールを担当する

Template : 画面に表示されるWebページの作成を担当する

アプリケーション開発の流れ

  1. プロジェクトを立ち上げる

    2. django-admin startproject project_name

  2. アプリケーションを立ち上げる

    3. django-admin startapp app_name or python manage.py startapp app_name

  3. 「setting.py」を編集し、アプリケーションを追加する

  4. 「models.py」を編集し、データベースを定義する

  5. マイグレーションファイルを作成し、マイグレーションを実行する

    6. python manage.py makemigrations

    python manage.py migrate

  6. 「admin.py」を編集し、データベースを管理可能な状態にする

  7. 「templatesディレクトリ」にhtmlファイルを配置する

  8. 「views.py」で関数を定義する

  9. 「urls.py」を編集する

  10. 実行

    11. python manage.py runserver

トップページの作成

説明

  1. プロジェクト名と同じ名前のディレクトリ内の「urls.py」を開く
  2. from app_name import viewsとpath('', views.index, name='index')を追加する
  3. djappディレクトリに「views.py」を作成する
  4. 「views.py」を編集する
  5. 上層のapp_nameディレクトリ内に「templates」ディレクトリを作成する
  6. 「templates」ディレクトリに「app_name」ディレクトリを作成する
  7. 「app_name」ディレクトリに「index.html」を作成する
  8. 下層のapp_nameの「setting.py」にimport osを追加した後、[TEMPLATES]内の[DIRS]を[os.path.join(BASE_DIR, 'templates')]にする
