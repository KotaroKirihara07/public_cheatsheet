Java

セットアップ

2025/06/15

このページではJava環境のセットアップ方法についての説明を行う。

JDKのインストール

  1. https://jdk.java.net/にアクセス
  2. 「Ready for use」右のversionを選択する
  3. 対応OS右横のファイルを選択する (Windowsはzip, Linuxはtar.gz)
  4. 「C:\」で展開する
  5. ユーザの環境変数に変数名「JAVA_HOME」、変数値「C:\jdk-?.?.?」とした環境変数を追加する
  6. 「%JAVA_HOME%\bin」というPathを追加する

コードの実行 (for コマンドプロンプト)

  1. 「Hello.java」というファイルを作成 (クラス名はHello)
  2. コマンドプロンプトでディレクトリの移動
  3. javac Hello.java でコンパイル
  4. java Hello で実行

プロジェクトの開始 (for VSCode)

#JDKの設定

  1. Javaの拡張機能 (Extension Pack for Java)をインストール
  2. 「JAVA PROJECTS」のプルダウンにある「Create new javaproject」
  3. 開発環境とプロジェクト名と保存先を選択する
  4. settings.jsonを"java.jdt.ls.java.home":"C:\\jdk-17.0_3"とする

#ビルドツールを使用しない場合

  1. 「View」→「Command Paltte」→「Java: Create Java Project」→「No build tools」
  2. ディレクトリを選択する
  3. プロジェクト名を決定する
  4. srcディレクトリ内にパッケージディレクトリを作成する
  5. パッケージディレクトリ内に「*.java」というファイルを作成する

#ビルドツールを使用する場合

  1. 「View」→「Command Paltte」→「Java: Create Java Project」→「Mavenプロジェクト/Gradleプロジェクト」
  2. ディレクトリを選択する
  3. プロジェクト名を決定する

プロジェクトの開始 (for Eclipse)

  1. 「workspace」というディレクトリを作成する
  2. 「project」という新規プロジェクトを作成する (場所はworkspaceの直下)
  3. 「project」を右クリックして新規パッケージを作成する
  4. 「project」を右クリックして新規クラスを作成する

＊「ウィンドウ」→「ビューの表示」→「その他」→「Java」→「パッケージ・エクスプローラー」でプロジェクトやパッケージやクラス階層を見ることができる。

プロジェクトの構成

作成したプロジェクトには3つのディレクトリ( 「.setting」、「bin」、「src」 )と2つのファイル( 「.classpath」、「.project」 )がある。

「.setting」にはEclipse内で作業したプロジェクトに対する設定に関するファイルが存在する。

「bin」はsrcディレクトリに格納されたJavaソースファイルをコンパイルしたクラスファイル(*.class)の出力先ディレクトリ。

「src」はJavaソースファイル(*.java)を格納するためディレクトリ。

「.classpath」はプロジェクトで指定したクラスパス(javaコマンドの-cpや環境変数classpathに該当)を管理するファイル。

「.project」はプロジェクト名などのプロジェクト固有の情報を管理するファイル。

パッケージとは

パッケージとは、関係のあるクラスをひとまとめにするための機能である。 すべてのクラスは必ず1つのパッケージに属する。 また、1つのパッケージ内で同名のクラスが存在することはできない。

パッケージを階層化させたい場合は、階層ごとにドットで区切って命名する。 例えば、パッケージ名を「java.test.pkg1」や「java.test.pkg2」とすることができる。

同一パッケージ内にあるクラスでは互いに「import ---」としなくても呼び出すことができる。 「java.test.pkg1」のとあるクラス(C1)から「java.test.pkg2」のとあるクラス(C2)を呼び出すには、 C1クラス内で「import java.test.pkg2.C2」として、C2を呼び出すことができるようになる。

ビルドツール

Javaのビルドツールとは、Javaプロジェクトのコンパイル、テスト、パッケージング、デプロイメントなど、ビルドプロセスを自動化するツール。

Apache Ant

Javaで記述されたビルドツールで、XMLベースの構成ファイルでタスクを定義する。

Apache Maven (メイヴン)

Javaプロジェクトのビルドと管理に広く使用されるツールで、「pom.xml」でプロジェクト設定を管理する。

#インストール

  1. 「C:\maven」に必要なファイルを...
  2. 環境変数に「C:\maven\bin」を追加します
  3. mvn -version

Gradle (グレイドル)

Java, Scala, Androidなど、複数の言語とプラットフォームをサポートするビルドツールで、柔軟性と拡張性に優れている。「build.gradle」にプロジェクトの設定を記述する。
