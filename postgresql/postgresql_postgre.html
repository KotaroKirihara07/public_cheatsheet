PostgreSQLのセットアップ

2024/05/15

このページではPostgreSQLのセットアップについての説明を行う。

インストール (Windows)

  1. 公式サイト https://www.postgresql.org/ へ移動
  2. downloadからOS選択
  3. インストールディレクトリの選択
  4. データを保存するディレクトリの選択
  5. 管理者ユーザのパスワードを設定する
  6. ポート番号の設定
  7. ロケールを設定する
  8. 設定ファイル(postgresql.conf)を書き換える

※ユーザ名がアスキーコードでない場合(漢字、ひらがな、カタカナ)はインストールに失敗するらしい。

インストール (Linux)

  1. sudo apt install -y postgresql postgresql-contrib

起動 (Windows)

  1. 管理者権限でコマンドプロンプトを開く
  2. net start postgresql-x64-16
  3. C:\PostgreSQL\16\binのパスを通す
  4. psql -U postgresと入力する (パスを通していない場合はC:\PostgreSQL\16\bin\psql.exe -U postgres)
  5. 管理者用パスワードを入力する
  6. データベースを作成する
  7. \c database_nameとして、データベースに接続する
  8. テーブルを作成する
  9. \qと入力して終了する
  10. net stop postgresql-x64-16

起動 (Linux)

  1. systemctl start postgresql
  2. sudo -u postgres psql
  3. \qと入力して終了する
  4. systemctl stop postgresql

ドライバ

ドライバとはアプリケーションとデータベースを接続させるためのプログラム。

ODBC (Open DataBase Connectivity) : MS社が1992年に発表したDBMSへの接続方法の規格であり、現在の業界標準となっている。

JDBC (Java DataBase Connectivity) : ODBCを参考にJavaアプリケーションからDBMSへの接続方法の規格。
